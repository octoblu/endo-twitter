Any attempt that would result in duplication will be blocked, resulting in a 403 error. Therefore, a user cannot submit the same status twice in a row.</p><p>While not rate limited by the API a user is limited in the number of Tweets they can create at a time. If the number of updates posted by the user reaches the current allowed limit this method will return an HTTP 403 error.</p><p><strong>About Geo</strong></p><ul><li>Any geo-tagging parameters in the update will be ignored if <code>geo_enabled</code> for the user is false (this is the default setting for all users unless the user has enabled geolocation in their settings)</li><li>The number of digits passed the decimal separator passed to <code>lat</code>, up to 8, will be tracked so that the <code>lat</code> is returned in a status object it will have the same number of digits after the decimal separator.</li><li>Please make sure to use to use a decimal point as the separator (and not the decimal comma) for the latitude and the longitude - usage of the decimal comma will cause the geo-tagged portion of the status update to be dropped.</li><li>For JSON, the response mostly uses conventions described in <a href="http://geojson.org/">GeoJSON</a>. Unfortunately, the geo object has coordinates that Twitter renderers are <strong>reversed</strong> from the GeoJSON specification (GeoJSON specifies a longitude then a latitude, whereas we are currently representing it as a latitude then a longitude. Our JSON renders as: <code>"geo": { "type":"Point", "coordinates":[37.78217, -122.40062] }</code></li><li>The coordinates object is replacing the geo object (no deprecation date has been set for the <code>geo</code> object yet) &#8212; the difference is that the coordinates object, in JSON, is now rendered correctly in GeoJSON.</li><li>If a <code>place_id</code> is passed into the status update, then that place will be attached to the status. If no <code>place_id</code> was explicitly provided, but <code>latitude</code> and <code>longitude</code> are, we attempt to implicitly provide a place by calling <a href="/rest/reference/get/geo/reverse_geocode">geo/reverse_geocode</a>.</li><li>Users will have the ability, from their settings page, to remove all the geotags from all their tweets en masse. Currently we are not doing any automatic scrubbing nor providing a method to remove geotags from individual tweets.</li></ul></div></div></div></div><div class="Node-content-field Node-apiDocsUrl"><h3>Resource URL</h3><code><div class=" Field FieldName-fieldDocResourceUrl FieldType-text FieldLabel-hidden "><div class="Field-items"><div class="Field-items-item even " >https://api.twitter.com/1.1/statuses/update.json</div></div></div></code></div><div class="Node-content-field Node-apiDocsResourceInfomation"><h3>Resource Information</h3><div class=" Field FieldName-fieldDocResponseFormats FieldType-listText FieldLabel-above "><div class="Field-label" >Response formats</div><div class="Field-items"><div class="Field-items-item even " >JSON</div></div></div><div class=" Field FieldName-fieldDocRequiredAuth FieldType-listText FieldLabel-above "><div class="Field-label" >Requires authentication?</div><div class="Field-items"><div class="Field-items-item even " >Yes (user context only)</div></div></div><div class=" Field FieldName-fieldDocRateLimited FieldType-listText FieldLabel-above "><div class="Field-label" >Rate limited?</div><div class="Field-items"><div class="Field-items-item even " >Yes</div></div></div></div><div class="Node-content-field Node-apiDocsParams"><div class=" Field FieldName-fieldDocParams FieldType-textLong FieldLabel-above "><h3 class="Field-label" > Parameters</h3><div class="Field-items"><div class="Field-items-item even " ><div class="parameter" id="api-param-status"><span class="param">status <span>required</span></span><p>The text of your status update, typically up to 140 characters. URL encode as necessary. <a href="/overview/t.co">t.co link wrapping</a> may affect character counts.</p><p>There are some <a href="https://support.twitter.com/articles/14020-twitter-for-sms-basic-features">special commands</a> in this field to be aware of. For instance, preceding a message with &#8220;D &#8221; or &#8220;M &#8221; and following it with a screen name can create a direct message to that user if the relationship allows for it.</p></div><div class="parameter" id="api-param-in_reply_to_status_id"><span class="param">in_reply_to_status_id <span>optional</span></span><p>The ID of an existing status that the update is in reply to.</p><p><strong>Note:</strong>: This parameter will be ignored unless the author of the tweet this parameter references is mentioned within the status text. Therefore, you must include <code>@username</code>, where <code>username</code> is the author of the referenced tweet, within the update.</p></div><div class="parameter" id="api-param-possibly_sensitive"><span class="param">possibly_sensitive <span>optional</span></span><p>If you upload Tweet media that might be considered sensitive content such as nudity, violence, or medical procedures, you should set this value to true. See <a href="https://support.twitter.com/articles/20169200">Media setting and best practices</a> for more context. Defaults to <code>false</code>.</p><p><strong>Example Values</strong>: <code>true</code></p></div><div class="parameter" id="api-param-lat"><span class="param">lat <span>optional</span></span><p>The latitude of the location this tweet refers to. This parameter will be ignored unless it is inside the range -90.0 to +90.0 (North is positive) inclusive. It will also be ignored if there isn&#8217;t a corresponding <code>long</code> parameter.</p><p><strong>Example Values</strong>: <code>37.7821120598956</code></p></div><div class="parameter" id="api-param-long"><span class="param">long <span>optional</span></span><p>The longitude of the location this tweet refers to. The valid ranges for longitude is -180.0 to +180.0 (East is positive) inclusive. This parameter will be ignored if outside that range, if it is not a number, if <code>geo_enabled</code> is disabled, or if there not a corresponding <code>lat</code> parameter.</p><p><strong>Example Values</strong>: <code>-122.400612831116</code></p></div><div class="parameter" id="api-param-place_id"><span class="param">place_id <span>optional</span></span><p>A <a href="/overview/api/places">place</a> in the world.</p><p><strong>Example Values</strong>: <code>df51dec6f4ee2b2c</code></p></div><div class="parameter" id="api-param-display_coordinates"><span class="param">display_coordinates <span>optional</span></span><p>Whether or not to put a pin on the exact coordinates a tweet has been sent from.</p><p><strong>Example Values</strong>: <code>true</code></p></div><div class="parameter" id="api-param-skip_user"><span class="param">trim_user <span>optional</span></span><p>When set to either <code>true</code>, <code>t</code> or <code>1</code>, each Tweet returned in a timeline will include a user object including only the status authors numerical ID. Omit this parameter to receive the complete user object.</p><p><strong>Example Values</strong>: <code>true</code></p></div><div class="parameter" id="api-param-media_ids"><span class="param">media_ids <span>optional</span></span><p>A list of <code>media_ids</code> to associate with the Tweet. You may include up to 4 photos or 1 animated GIF or 1 video in a Tweet. See <a href="https://dev.twitter.com/rest/public/uploading-media">Uploading Media</a> for further details on uploading media.</p><p><strong>Example Values</strong>: <code>471592142565957632</code></p></div></div></div></div></div><div class="Node-content-field Node-oauthGenerator"><h3>OAuth Signature Generator</h3><p><a href="https://twitter.com/login">Sign in</a> to see a list of your registered applications.</p></div><div class="Node-content-field Node-apiDocsRequest"><h3>Example Request</h3><code>POST
